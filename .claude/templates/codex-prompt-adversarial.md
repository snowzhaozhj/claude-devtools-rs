# codex 对抗式验证 prompt 模板

## 何时用

diff 涉及并发/状态机/缓存/错误恢复/配置组合/async 生命周期时（触发点 #4）。

## prompt 骨架

```
背景：[一句话讲改动目的]

改动涉及的机制：[并发原语 / 状态转移 / 缓存失效 / retry 逻辑 / 配置交互 / async spawn-drop]

关键代码：
[贴精简 diff 或核心函数，标注不变量]

已有保护措施：
- [锁 X 保护 Y]
- [test Z 覆盖了 W]

请你作为攻击者构造：
1. 3 个具体事件序列让系统进入非预期状态
2. 每个序列标注：关键时间点 + await 点 / 锁释放窗口 + 破坏的不变量
3. 忽略类型系统已保证的不变量（&mut 独占、lifetime）
4. 对每个序列建议一个能防住它的测试

约束：中文，每个序列 ≤ 5 步，总共 ≤ 600 字
```

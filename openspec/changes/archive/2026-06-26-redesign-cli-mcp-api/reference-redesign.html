一、设计原则

渐进式披露，每层 schema 稳定

list_sessions（选 session）→ get_session（看全貌）→ get_turn（看调用链）→ get_step_output（看原文）。
每个工具永远返回同一结构，不因参数组合变形。

统一分页：total + nextCursor（MCP 协议约定）

跟 MCP 协议一致：nextCursor 存在 = 还有下一页，不存在 = 到底了。
hasMore 和 returned 是冗余字段（nextCursor 存在即 hasMore，items.length 即 returned），不加。
分页字段只有 2 个（total + nextCursor），直接写在各 response struct 里，不做泛型/组合抽象。

CLI 和 MCP 数据参数完全一致

CLI 额外只有终端渲染 flags（--format / --json / --no-truncate / --raw），不存在"CLI-only 数据参数"。

API 自闭环

截断的数据都有 escape hatch 取回全文（get_step_output），不依赖外部工具（如 Read）。

五、删除参数总览

反模式类（为省 token 加的有害参数）

参数原工具问题
content_modeget_session_chunks默认 omit 逼迫多次调用。turn 模型 + 内置截断替代
includeget_sessionAI 不知道要 opt-in 什么。turn 视图自带 tools/metrics
filterget_session_chunks返回数据已标记 isError/type，客户端一行过滤
max_chunksget_session_chunks配合 content_mode 的人为限制
grep_contextget_session_chunkschunk 不是语义单元，按 chunk 切上下文断裂。turn 是语义单元

合并/替代类

参数原工具去向
range / tail / cursorget_session_chunks→ get_turn 的 turn index + cursor
get_session + get_session_chunks2 tools→ get_session（compact）+ get_turn（detail）+ get_step_output（全文）

客户端可做类

参数原工具理由
branchlist_sessions返回含 gitBranch，20 条过滤成本 ≈ 0
is_ongoinglist_sessions返回含 isOngoing，一行过滤
limitlist_sessions / search默认 20 + cursor
group_bylist_sessions / get_stats对已返回数据做 in-memory 分桶，客户端 reduce 搞定

冗余类

参数原工具理由
projectget_session / get_session_chunkssession ID 全局唯一，服务端自动解析

八、实现路径

所有改动锁在 CLI/MCP 专属层。桌面端依赖 cdt-api/cdt-cli 但渲染会话走 Tauri IPC → LocalDataApi，不 link cdt-query，按构造零影响。

层桌面端是否共享变化
cdt-core / cdt-parse / cdt-analyze共享不变。semantic_steps + tool_executions + AIChunk.subagents 已有
cdt-api::get_session_detail共享不改 hot path，只消费其已有输出（含已嵌入的 subagent messages）
cdt-query::EngineCLI/MCP 专属Turn 构建 + subagent 解析（含 lazy 反查索引）的主场。chunks 配对 + semantic_steps 骨架 + tool merge + answer 判定 + 截断
cdt-cli/src/view.rsCLI/MCP 专属新增 TurnCompact / Step / StepOutput / SubagentStep view structs
cdt-cli/src/mcp/mod.rsCLI/MCP 专属删 get_session_chunks → 新增 get_turn / get_step_output → 重定义 get_session → 精简参数
cdt-cli/src/main.rsCLI/MCP 专属新增 turn / step-output 子命令 + session 默认 turns 输出 + --raw 保留 chunk 路径
关键实现点
① CLI/MCP 直接调 cdt_parse + cdt_analyze 拿完整数据，不走 IPC 的 OMIT_TOOL_OUTPUT
② persisted-output 占位符：构建 turn 时透明回读真实文件内容
③ semantic_steps + AIChunk.subagents 已在数据层，当前 view.rs 没暴露——turn 视图直接读取
④ subagent 解析放 cdt-query::Engine（CLI/MCP 专属），桌面端不 link，扩展零影响其性能
⑤ Turn 构建逻辑与前端 displayItemBuilder.ts 同源，可考虑后续下沉到 cdt-analyze 共用

九、边界 Case

场景行为
会话 >20 turnsget_session 返回最近 20 + nextCursor
turn >50 stepsget_turn 返回前 50 + nextCursor
tool output ≥ 5KB截前 2000 字 + outputTruncated: true + outputBytes。get_step_output 取全文
thinking不截断（实测 p95 = 4.4 KB，由 step 分页控制总体积）
AI turn 无 answer（被打断）answer: null，steps 保留到中断点
连续多个 AI chunk合并为同一个 turn 的 steps
Compact chunk不产 turn；get_session 顶层 compactions: [{afterTurn, summary}]
Subagent spawnstep: { type: "subagent", name: "Explore", description: "...", subagentSessionId: "agent-abc", stepsCount: 42 }。AI 用 subagentSessionId 调 get_session/get_turn 递归钻取（A1）
subagent 解析subagent JSONL 在 <project>/<父session>/subagents/，cdt-query::Engine 建 lazy 反查索引（subagentSessionId → 父）。CLI/MCP 专属，桌面端不付代价
persisted-output 占位符服务端透明回读文件内容
grep 匹配范围匹配完整 steps 内容（含 thinking/tool input/output），返回 matchedIn 标注
grep + cursor先 grep 过滤，结果集按 20/page 分页
search 同 session 多 turn 命中每个 turn 独立一条 result，平铺返回
since + until闭区间，在分页之前执行
session = "latest"get_session / get_turn 均支持
step index 越界get_step_output 返回错误
turn index 越界get_turn 返回错误

十、设计决策记录

#决策理由
D1get_session 和 get_turn 拆两个工具同一接口返回不同 schema 对 AI 不友好
D2加 get_step_output 工具API 自闭环，不依赖外部 Read（Bash 输出/MCP 返回无文件路径可读）
D3tools 按工具类型聚合，含 per-type errorCountper-call 列表无上限；per-type 天然有界（5-15 种），且 AI 不钻取即知哪种工具出错
D4分页用 total + nextCursor，不加 hasMore/returned跟 MCP 协议约定一致；nextCursor 存在即 hasMore，items.length 即 returned
D5分页字段直接写，不做泛型/组合抽象只有 2 个字段，泛型 fit 不了复合响应，flatten 有 serde 坑
D6question/answer 不截断实测 UserChunk content 排除系统注入后大多 10-200 B，answer 大多 100-2000 B
D7thinking 不截断实测 p95 = 4.4 KB，由 step 分页（50/page）控制总体积
D8tool output ≥5KB 截断前 2000 字大 output 主要是 Read 文件内容，单个可达几十 KB。get_step_output 取全文
D9get_turn steps 从前往后answer 已在顶层，drill-down 目的是看因果推理链，从头更自然
D10search 返回 turn 级命中，平铺AI 拿到 turnIndex 可直接 get_turn 钻取，不需要中间跳转
D11grep 匹配完整 steps 内容 + matchedIn 标注只搜 question/answer 会漏掉工具调用里出现的内容
D12filesTouched 在 list_sessions，userIntents 在 get_sessionfilesTouched 小（800B/session）且提供"实际修改"语义（search 覆盖不了）；userIntents 大（3KB）且与 title 重叠
D13删掉顶层 errors 列表被 turn 模型覆盖：compact tools[].errorCount + get_turn steps isError
D14删 branch/is_ongoing/limit/group_by/filter/content_mode/include 等分别属于"客户端可做"/"反模式"/"冗余"，详见§五
D15subagent 当独立 session 暴露（A1），step 放 subagentSessionId，复用 get_session/get_turn 递归钻取subagent 本就是有 session_id 的独立会话，统一心智不加新工具。数据已随父加载，反查索引 lazy 化
D16扩展锁在 cdt-query::Engine + cdt-cli view 层，不碰 cdt-core/cdt-analyze 及 cdt-api hot path桌面端不 link cdt-query，按构造零性能影响；共享层只消费已有输出不改算法
